# Diagnose failure mode before editing

Before rewriting, identify the main problem:

- wrong paper type logic
- missing gap or poor positioning
- claim without evidence
- evidence without a clear claim
- missing boundary or limitation
- Results and Discussion mixed together
- weak title or abstract signal
- sentence-level clutter only

Prioritize fixes in this order:

`paper type -> section job -> paragraph logic -> claim/evidence/boundary -> sentence polish`

Do not sentence-polish a draft whose section job is wrong. Surface the structural problem first, then polish.

## Journal Compliance Audits

When polishing for a target journal, load
`../../../windenergy-submission/references/journal-profiles.md` and apply the
matching slug. Use `common-18` if no journal is named.

### Length audit

If the section being polished pushes the manuscript toward a target limit, flag
the risk and name the affected profile item.

### Reference count audit

Check the target profile before applying any reference-count limit. Do not add,
prune, or renumber references without author approval.

### English variant consistency

Use either American or British English consistently. When polishing:

| Check | American | British |
|-------|----------|---------|
| -ize/-ise endings | optimize, analyze | optimise, analyse |
| -or/-our endings | color, behavior | colour, behaviour |
| -er/-re endings | center, meter | centre, metre |
| Double consonants | modeled, labeling | modelled, labelling |
| Past tense | learned, spelled | learnt, spelt |

Pick one variant and enforce it throughout the section. Flag any inconsistencies.

### Scope alignment

When polishing, verify the text connects to the target journal's scope. For
wind and AI manuscripts, the safest framing ties method claims to renewable
generation, forecasting, integration, operation, control, or decision support.

### Highlights audit (if polishing highlights)

Each bullet should be max 85 characters including spaces unless the target
profile says otherwise. Check character count after polishing.

### Math formulae

When polishing sections containing equations, ensure:
- Equations are editable text (not images)
- Simple formulae inline with normal text
- Solidus (/) for small fractions, not horizontal lines
- Variables in italics
- Powers of e written as exp(...)
- Display equations numbered consecutively
